# Contribuindo para SyntaxWear

Obrigado por se interessar em contribuir com o projeto SyntaxWear! Este documento fornece diretrizes e instruções para contribuir.

## Como Contribuir

### 1. Fork o Projeto
Faça um fork do repositório para sua conta GitHub.

### 2. Crie uma Branch
```bash
git checkout -b feature/nome-da-feature
```

### 3. Faça Commits Significativos
```bash
git commit -m "Add: descrição clara da mudança"
```

### 4. Push para a Branch
```bash
git push origin feature/nome-da-feature
```

### 5. Envie um Pull Request
Abra um Pull Request com uma descrição clara das mudanças.

## Padrões de Código

### HTML
- Use semântica HTML5
- Indente com 4 espaços
- Sempre feche tags corretamente

### CSS
- Use variáveis CSS (custom properties)
- Organize por seções com comentários
- Mantenha a especificidade baixa
- Use flexbox e grid quando possível

### JavaScript
- Use ES6+
- Mantenha funções pequenas e focadas
- Adicione comentários para lógica complexa
- Use nomes descritivos para variáveis e funções

## Convenção de Commits

- `Add:` para novas funcionalidades
- `Fix:` para correção de bugs
- `Change:` para mudanças em código existente
- `Remove:` para remoções
- `Refactor:` para reorganização de código
- `Docs:` para documentação

## Reporte de Bugs

Ao relatar bugs, inclua:
- Descrição clara do problema
- Passos para reproduzir
- Comportamento esperado
- Comportamento atual
- Screenshots (se aplicável)
- Informações do seu ambiente (navegador, SO, etc)

## Sugestões de Features

Sinta-se livre para sugerir novas funcionalidades através de issues. Descreva:
- O que você gostaria de adicionar
- Por que isso seria útil
- Possíveis implementações

## Perguntas?

Se tiver dúvidas, abra uma issue ou entre em contato através do email: contato@syntaxwear.com

Obrigado por contribuir! 🚀
